# Entrypoint Bindings Plan

This table records the DeepBook Predict entrypoints the project expects to inspect or bind. DeepVol is now the primary Predict-native structured product layer: UP, DOWN, and RANGE are advanced primitives, and BTC MOVE Receipt is the primary composed product. Binary-leg entrypoints are promoted from diagnostic-only work to DeepVol-critical future binding work. It is not final SDK code; every exact generated binding, PTB object argument order, pure argument encoding, return mapping, and event field is `MUST CONFIRM BEFORE CODING`.

Phase 1A added read-only public server discovery and SDK server helpers only. No entrypoint binding, wallet action, or PTB write path was validated. Public server data can support oracle selection and quote asset/vault context, but cannot validate `mint_range`, binary `mint`, or redeem PTB construction.

DeepVol-specific binary requirements are detailed in [DEEPVOL_BINARY_LEG_INTEGRATION.md](./DEEPVOL_BINARY_LEG_INTEGRATION.md), the controlled mint round is recorded in [DEEPVOL_BINARY_MINT_TESTNET_VALIDATION.md](./DEEPVOL_BINARY_MINT_TESTNET_VALIDATION.md), and the product-layer boundary is documented in [DEEPVOL_PRIMITIVES_AND_RECEIPTS.md](./DEEPVOL_PRIMITIVES_AND_RECEIPTS.md). Binary quote/read/preflight and one controlled two-leg binary mint have Testnet evidence, but binary redeem builders are not production-complete.

Phase 1B adds browser-wallet-only scaffolding for Predict Account setup. Phase 1B-Verify validated the no-argument `create_manager` move call, event/object-change manager ID recovery, a gated local signer `deposit<DUSDC>` PTB, and known-manager public server summary readback. Browser wallet manual validation and general owner discovery remain pending. See [PREDICT_MANAGER_FLOW.md](./PREDICT_MANAGER_FLOW.md) and [PREDICT_MANAGER_TESTNET_VALIDATION.md](./PREDICT_MANAGER_TESTNET_VALIDATION.md) for the current flow and validation artifacts.

DeepVol-3B adds Route B entrypoints in `move/deepvol` for `series::create_series`, `series::deactivate_series`, `vault::create_protocol_vault<Quote>`, `vault::withdraw_protocol_fees<Quote>`, `receipt::buy_move_receipt<Quote>`, and `receipt::mark_receipt_settled`. DeepVol-4 records the manual Testnet package publish and creates the shared `ProtocolVault<DUSDC>`; package, protocol vault, AdminCap, UpgradeCap, and publisher IDs are now configured in [DEEPVOL_TESTNET_PUBLISH_RESULT.md](./DEEPVOL_TESTNET_PUBLISH_RESULT.md). No deployed `VolSeries`, `MoveReceipt`, or `buy_move_receipt<DUSDC>` execution is implied yet.

Phase 1C added quote-only and gated mint validation helpers. The first 2026-05-16 quote-only run selected an active BTC oracle at runtime and attempted `range_key::new` plus `predict::get_range_trade_amounts`, but the selected range aborted in `pricing_config::quote_spread_from_fair_price` with abort code `1`. Phase 1C-fix added a quoteability scanner; it treats ask-bounds `null` as diagnostic, identifies abort code `1` as invalid boundary fair price, verifies successful `(mint_cost, redeem_payout)` decoding, and still blocks mint because the selected quote returned `mint_cost = 0`. Phase 1C-fix2 added quantity sweep, safe return-decoding diagnostics, binary quote sanity checks, and wider/asymmetric range candidates; it found positive official range quotes and nonzero binary quotes, then the real mint path failed with code `7` in `predict::assert_mintable_ask`. Phase 1C-fix3 identifies code `7` as `EAskPriceOutOfBounds`, validates onchain ask-bounds inspection, and adds full `mint_range<DUSDC>` preflight gating; the bounded mintability scan found `0` full preflight successes across `40` preflight attempts. Phase 1C-debug inspected `deepbookv3-predict-package/predict` and confirmed that `get_range_trade_amounts` prices current state while `mint_range` inserts range exposure, refreshes oracle risk, recomputes ask, and then calls `assert_mintable_ask`. See [RANGE_MINT_TESTNET_VALIDATION.md](./RANGE_MINT_TESTNET_VALIDATION.md), [RANGE_QUOTEABILITY_INVESTIGATION.md](./RANGE_QUOTEABILITY_INVESTIGATION.md), [RANGE_QUOTE_UNITS_AND_DECODING.md](./RANGE_QUOTE_UNITS_AND_DECODING.md), [MINTABILITY_PREFLIGHT_AND_ASK_BOUNDS.md](./MINTABILITY_PREFLIGHT_AND_ASK_BOUNDS.md), and [MINTABILITY_SOURCE_ANALYSIS.md](./MINTABILITY_SOURCE_ANALYSIS.md). Phase 1D-2 validated `predict::redeem_range<DUSDC>` with direct pre/post `range_position`, full preflight, one live Testnet redeem, and `RangeRedeemed` parsing; see [RANGE_REDEEM_TESTNET_VALIDATION.md](./RANGE_REDEEM_TESTNET_VALIDATION.md). Phase 3A/B selects Route B: creator follow transactions target the RangePilot wrapper, which internally calls DeepBook Predict `mint_range<DUSDC>` after Strategy/fee validation. Phase 3D replaces direct platform-recipient routing with `ProtocolVault<T>` + `AdminCap`: creator fee transfers to the creator, platform fee deposits into `ProtocolVault<T>`, and SDK builders require wrapper package ID, protocol vault ID, quote preview, and full mint preflight before building. Phase 3E-postpublish records the manually published wrapper package, AdminCap, UpgradeCap, and shared `ProtocolVault<DUSDC>`. Phase 3F validates the first wrapper follow with digest `997Yu78xbiM57fbJUsVk1eKURcbt8SXdi7Ypb1H74HEB`. The direct DeepBook Predict entrypoints remain the source of truth for quote, preflight, diagnostics, and protocol behavior.
