# Entrypoint Bindings Plan

This table records the DeepBook Predict entrypoints RangePilot expects to inspect or bind during the protocol integration spike. It is not final SDK code; every exact generated binding, PTB object argument order, pure argument encoding, return mapping, and event field is `MUST CONFIRM BEFORE CODING`.

Phase 1A added read-only public server discovery and SDK server helpers only. No entrypoint binding, wallet action, or PTB write path was validated. Public server data can support oracle selection and quote asset/vault context, but cannot validate `mint_range` or `redeem_range` PTB construction.

Phase 1B adds browser-wallet-only scaffolding for Predict Account setup. Phase 1B-Verify validated the no-argument `create_manager` move call, event/object-change manager ID recovery, a gated local signer `deposit<DUSDC>` PTB, and known-manager public server summary readback. Browser wallet manual validation and general owner discovery remain pending. See [PREDICT_MANAGER_FLOW.md](./PREDICT_MANAGER_FLOW.md) and [PREDICT_MANAGER_TESTNET_VALIDATION.md](./PREDICT_MANAGER_TESTNET_VALIDATION.md) for the current flow and validation artifacts.

Phase 1C added quote-only and gated mint validation helpers. The first 2026-05-16 quote-only run selected an active BTC oracle at runtime and attempted `range_key::new` plus `predict::get_range_trade_amounts`, but the selected range aborted in `pricing_config::quote_spread_from_fair_price` with abort code `1`. Phase 1C-fix added a quoteability scanner; it treats ask-bounds `null` as diagnostic, identifies abort code `1` as invalid boundary fair price, verifies successful `(mint_cost, redeem_payout)` decoding, and still blocks mint because the selected quote returned `mint_cost = 0`. Phase 1C-fix2 added quantity sweep, safe return-decoding diagnostics, binary quote sanity checks, and wider/asymmetric range candidates; it found positive official range quotes and nonzero binary quotes, then the real mint path failed with code `7` in `predict::assert_mintable_ask`. Phase 1C-fix3 identifies code `7` as `EAskPriceOutOfBounds`, validates onchain ask-bounds inspection, and adds full `mint_range<DUSDC>` preflight gating; the bounded mintability scan found `0` full preflight successes across `40` preflight attempts. Phase 1C-debug inspected `deepbookv3-predict-package/predict` and confirmed that `get_range_trade_amounts` prices current state while `mint_range` inserts range exposure, refreshes oracle risk, recomputes ask, and then calls `assert_mintable_ask`. See [RANGE_MINT_TESTNET_VALIDATION.md](./RANGE_MINT_TESTNET_VALIDATION.md), [RANGE_QUOTEABILITY_INVESTIGATION.md](./RANGE_QUOTEABILITY_INVESTIGATION.md), [RANGE_QUOTE_UNITS_AND_DECODING.md](./RANGE_QUOTE_UNITS_AND_DECODING.md), [MINTABILITY_PREFLIGHT_AND_ASK_BOUNDS.md](./MINTABILITY_PREFLIGHT_AND_ASK_BOUNDS.md), and [MINTABILITY_SOURCE_ANALYSIS.md](./MINTABILITY_SOURCE_ANALYSIS.md). Phase 1D-2 validated `predict::redeem_range<DUSDC>` with direct pre/post `range_position`, full preflight, one live Testnet redeem, and `RangeRedeemed` parsing; see [RANGE_REDEEM_TESTNET_VALIDATION.md](./RANGE_REDEEM_TESTNET_VALIDATION.md). Phase 3A/B selects Route B: future creator follow transactions target the RangePilot wrapper, which internally calls DeepBook Predict `mint_range<DUSDC>` after Strategy/fee validation. The direct DeepBook Predict entrypoints remain the source of truth for quote, preflight, diagnostics, and protocol behavior.
