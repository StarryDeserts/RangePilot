# Entrypoint Bindings Plan

This table records the DeepBook Predict entrypoints RangePilot expects to inspect or bind during the protocol integration spike. It is not final SDK code; every exact generated binding, PTB object argument order, pure argument encoding, return mapping, and event field is `MUST CONFIRM BEFORE CODING`.

Phase 1A added read-only public server discovery and SDK server helpers only. No entrypoint binding, wallet action, or PTB write path was validated. Public server data can support oracle selection and quote asset/vault context, but cannot validate `mint_range` or `redeem_range` PTB construction.

Phase 1B adds browser-wallet-only scaffolding for Predict Account setup. Phase 1B-Verify validated the no-argument `create_manager` move call, event/object-change manager ID recovery, a gated local signer `deposit<DUSDC>` PTB, and known-manager public server summary readback. Browser wallet manual validation and general owner discovery remain pending. See [PREDICT_MANAGER_FLOW.md](./PREDICT_MANAGER_FLOW.md) and [PREDICT_MANAGER_TESTNET_VALIDATION.md](./PREDICT_MANAGER_TESTNET_VALIDATION.md) for the current flow and validation artifacts.

Phase 1C added quote-only and gated mint validation helpers. The 2026-05-16 quote-only run selected an active BTC oracle at runtime and attempted `range_key::new` plus `predict::get_range_trade_amounts`, but mint remained blocked because ask-bounds returned `null` and quote devInspect aborted in `pricing_config::quote_spread_from_fair_price` with abort code `1`. See [RANGE_MINT_TESTNET_VALIDATION.md](./RANGE_MINT_TESTNET_VALIDATION.md).
